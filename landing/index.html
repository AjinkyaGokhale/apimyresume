A simple REST API

Everything the dashboard does is available over HTTP — ready for AI agents, n8n, and Zapier.

Authentication. Every request to /api/v1/* sends your key in an X-API-Key header. Base URL: https://your-domain/api/v1. Endpoints marked owner require the dashboard owner session and are not available to API keys — agents can read bases and tailor child resumes, but never alter the base structure.

Base resumes

GET/basesList all base resumes
POST/basesCreate a base resume owner
GET/bases/{id}Get a single base
GET/bases/{id}/contentAI-optimised read of base content
PATCH/bases/{id}Update profile, skills & experience owner
PATCH/bases/{id}/experience/{entryId}/bulletsReplace one job's bullets owner
POST/bases/{id}/regenerate-childrenRe-render all child PDFs owner
DELETE/bases/{id}Delete a base owner

Tailored resumes

POST/resumesCreate a tailored child resume
POST/resumes/previewPreview a render without saving
GET/resumesList tailored resumes
GET/resumes/{id}Get a single resume
GET/resumes/{id}/pdfDownload the rendered PDF
PATCH/resumes/{id}Update overrides & re-render
POST/resumes/{id}/regenerateRe-render without changes
DELETE/resumes/{id}Delete a resume

Templates & API keys

GET/templatesList available templates
GET/templates/{id}/thumbnailTemplate preview image
GET/api-keysList API keys
POST/api-keysCreate a key (shown once)
DELETE/api-keys/{id}Revoke a key

Meta

GET/healthService health & render-pool status
GET/schemaFull machine-readable schema reference
Example — tailor a resume to a job 
$ curl -X POST https://your-domain/api/v1/resumes \
    -H "X-API-Key: amr_live_xxxx" \
    -H "Content-Type: application/json" \
    -d '{
      "base_id": "john-doe-xxxx",
      "company": "Acme Corp",
      "role": "Senior Backend Engineer",
      "overrides": {
        "keywords": ["Go", "Kubernetes"]
      }
    }'

→ { "id": "resume_xxx", "pdf_url": "/pdfs/resume_xxx_v1.pdf" }
